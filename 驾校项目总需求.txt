3. 详细功能性需求
结合业务场景，将系统功能划分为七大核心模块，各模块详细功能需求如下：
3.1 学员报名管理模块（核心模块）
本模块实现学员全流程线上报名与资格审核，替代传统线下报名模式，分为学员自主操作与系统、管理员审核两部分。
3.1.1 在线注册与信息填写
•支持学员手机号、验证码完成账号注册，设置登录密码，实现账号密码登录、短信快捷登录。
•学员登录后可在线填报个人基础信息，包含姓名、身份证号、性别、年龄、联系电话、居住地址、紧急联系人、报考准驾车型等必填信息。
•系统开启信息校验机制：身份证号格式、手机号格式、年龄合法性实时校验，避免无效信息录入。
•支持信息临时保存，学员可分次填写报名信息，无需一次性完成。
3.1.2 资质材料上传
•支持学员线上上传身份证正反面照片、个人一寸证件照、正规体检报告照片/扫描件。
•设置文件上传规则：支持JPG、PNG、PDF格式，单文件大小限制，避免超大文件上传；支持图片预览、重新上传、删除替换。
•上传材料自动留存备份，关联学员个人账号，支持后续查询、调取、打印。
3.1.3 系统自动资格初审
学员提交报名信息与材料后，系统自动触发初审机制，无需人工干预，审核维度如下：
•年龄审核：根据身份证号自动识别年龄，匹配对应准驾车型年龄标准（如C1驾照18周岁以上），不符合则初审驳回并提示具体原因。
•准驾车型资格审核：校验学员报考车型与年龄、资质匹配性，禁止跨资质报考。
•体检状态审核：校验体检材料是否上传、体检结果是否合格，无体检材料或体检不合格直接初审驳回。
3.1.4 管理员人工复审
•系统初审通过的报名单自动进入管理员复审列表，状态显示“待复审”。
•管理员可查看学员全部报名信息、上传材料、系统初审结果，人工核对信息真实性、材料完整性。
•支持审核通过、审核驳回两种操作，驳回需填写驳回原因，实时同步至学员端。
•学员可实时查看报名状态（待初审、待复审、审核通过、审核驳回）及驳回原因，支持修改信息后重新提交审核。
3.2 材料自动生成模块
学员报名审核全部通过后，系统自动抓取学员报名信息，生成标准化官方表单，实现无纸化材料管理。
3.2.1 自动生成核心表单
系统自动批量生成三类标准化文档，数据与学员报名信息实时同步、自动填充，无需人工手动填写：
•驾校正式报名表：包含学员个人信息、报考车型、报名时间、审核状态等完整信息。
•体检信息表：关联学员上传体检数据、体检结果、体检时间等信息。
•学员准考证：包含学员信息、报考车型、考试科目、考场信息、准考证编号等。
3.2.2 文档预览、导出与打印
•支持所有生成表单在线预览功能，学员与管理员可直接在系统内查看完整文档内容。
•支持一键导出PDF格式文件，文件命名自动关联学员姓名+报考车型，方便归档管理。
•支持一键连接打印机直接打印，适配A4标准纸张，格式标准化、无错乱。
•所有生成文档永久留存系统后台，支持随时调取、重复导出打印。
3.3 教练管理与分配模块
实现教练信息统一管理、档期管控、智能分配与学员绑定，解决教练分配混乱、资源匹配不合理问题。
3.3.1 教练信息管理
•管理员可新增、编辑、删除、查询教练信息，核心信息包含：姓名、性别、联系方式、准驾教学车型、从业年限、学员评分、可带学员数量、个人简介、头像。
•教练可自主登录系统维护个人简介、更新可预约档期，不可修改核心资质信息（由管理员管控）。
•系统自动统计教练综合评分、当前带教人数、历史带教总量、教学通过率等数据，形成教练档案。
•支持教练状态管控：正常在岗、暂停接单、离职，不同状态对应不同分配权限。
3.3.2 教练智能/手动分配
•自动分配：学员报名审核通过后，系统根据三大维度智能推荐最优教练：①学员报考车型与教练教学车型匹配；②教练当前档期空闲、带教人数未满；③教练综合评分优先，自动生成推荐列表。
•手动分配：管理员可根据实际运营需求，手动为学员指定对应车型的在岗教练，灵活调整分配结果。
3.3.3 学员教练绑定管理
•分配确认后，系统自动完成学员与教练一对一绑定，生成绑定记录，永久留存。
•学员端可实时查看绑定教练的姓名、联系方式、评分、档期、教学车型等信息。
•管理员支持解绑、重新绑定操作，适配教练调动、学员换教练场景，同时记录变更日志。
•教练端可查看所有已绑定学员的基础信息、报名进度、学习状态。
3.4 学习进度管理模块
全程记录学员学车全流程，实现学时管控、约课管理、学习阶段实时跟踪，确保学车流程合规有序。
3.4.1 学时与练课记录管理
•教练可根据学员实际练车情况，录入每次练车的学时、练车内容、练车时长、练车评价。
•系统自动累计学员总学时、各科目专项学时，实时更新学时数据，达到官方学时标准后自动标记达标。
•所有练课记录、学时记录可追溯，学员与管理员均可查询明细，支持导出统计。
3.4.2 线上约课与取消约课
•学员可查看绑定教练的空闲档期，自主选择练车时间、提交约课申请。
•支持学员自主取消未开始的约课，系统自动释放档期资源；临近课时限制取消，避免资源浪费。
•约课成功、取消约课均实时推送状态更新，同步展示在学员与教练端日程列表。
•教练可查看学员预约列表，合理安排教学工作。
3.4.3 学习阶段全流程跟踪
系统将学员学车流程划分为四个标准阶段，自动跟踪阶段进度，实时更新学员学习状态：
•科目一学习阶段：记录理论学习时长、刷题进度、学时达标状态。
•科目二训练阶段：记录场地练车学时、训练次数、技能掌握情况。
•科目三训练阶段：记录道路练车学时、上路训练次数。
•科目四学习阶段：记录理论学习进度、学时达标状态。
•单个阶段学时、训练达标后，系统自动更新阶段状态为“已完成”，解锁下一阶段学习与考试报名权限；未达标则无法报名对应科目考试。
3.5 考试管理模块
覆盖学员考试报名、成绩录入、补考管理、证件发放全流程，规范考试业务管理。
3.5.1 考试报名管理
•学员对应学习阶段学时达标后，系统自动开放该科目考试报名权限。
•学员可自主选择报考科目、考试时间、对应考场，提交考试报名申请。
•管理员审核报名资格（学时、学习阶段、报名信息），审核通过则报名生效，驳回则提示原因。
•系统记录所有考试报名记录，区分待审核、报名成功、报名驳回状态。
3.5.2 成绩录入与查询
•考试结束后，管理员可批量/单个录入学员各科考试成绩，标注合格/不合格状态。
•学员端可实时查询各科考试成绩、考试时间、考试状态。
•系统自动区分首考、补考成绩，单独记录存档。
3.5.3 补考与发证管理
•考试不合格学员，系统自动生成补考记录，支持管理员登记补考安排、补考次数。
•记录学员每科补考详情，包含补考时间、补考成绩、补考费用等信息。
•学员所有科目考试全部合格后，系统自动触发证件发放提醒，学员端可查看待发证状态。
•管理员可登记证件发放记录，标记已发证、未发证状态，完成学车闭环管理。
3.6 基础信息管理模块
为系统所有业务提供基础数据支撑，由管理员统一维护，确保数据统一、准确、可管控，支持新增、编辑、删除、查询、启用/禁用操作，具体管理内容如下：
•车型管理：录入C1、C2、B2等各类准驾车型，设置对应报考条件、学时标准、收费标准。
•科目管理：维护科目一、科目二、科目三、科目四基础信息，设置各科目学时要求、考试规则。
•教练管理：全局教练信息台账维护，包含资质、状态、评分、档期等基础数据。
•考场管理：录入合作考场名称、地址、可考科目、考试时间段等信息。
•费用标准管理：设置不同车型报名费、补考费、学时费等收费标准，统一收费规范。
3.7 统计分析模块
系统自动抓取全业务数据，进行多维度统计分析，以数据报表+可视化图表形式展示，支持数据导出，为运营决策提供支撑。
3.7.1 核心统计维度
•报名数据统计：按日、周、月、年统计新增报名人数、审核通过人数、驳回人数、各车型报名占比。
•考试数据统计：各科目考试通过率、挂科率、补考人数、各科成绩分布情况。
•教练数据统计：各教练带学员数量、总学时、学员评分、学员考试通过率、工作量排名。
3.7.2 数据展示与导出
•可视化图表展示：柱状图、折线图、饼图，直观呈现数据变化趋势、占比分布。
•支持自定义统计时间段，灵活查询指定周期内的业务数据。
•所有统计报表支持Excel格式导出、保存、打印，方便台账归档与数据分析。
业务流程：
4. 核心业务详细流程
4.1 学员在线报名全流程
学员进入系统 → 手机号注册并登录账号 → 填写个人基础信息与报考车型 → 上传身份证、证件照、体检报告材料 → 提交报名申请，状态变为【待系统初审】→ 系统自动校验年龄、准驾资质、体检状态，完成初审 → 初审通过进入【待管理员复审】，初审驳回直接反馈原因 → 管理员人工核对信息与材料，完成复审 → 复审通过：生成学员全套报名资料，进入待教练分配状态；复审驳回：反馈驳回原因，学员修改后可重新提交。
4.2 材料自动生成与打印流程
学员报名审核（初审+复审）全部通过 → 系统自动抓取学员报名数据 → 批量生成标准化报名表、体检表、准考证 → 学员/管理员可在线预览文档 → 支持一键PDF导出、本地保存 → 支持在线直接打印文档 → 系统后台自动归档所有生成材料，永久留存。
4.3 教练分配与绑定流程
学员报名审核通过，状态为【待分配教练】→ 系统根据报考车型匹配、教练空闲档期、教练评分三大条件，智能筛选并推荐最优教练列表 → 管理员查看推荐结果，可确认自动分配结果或手动更换教练 → 确认分配后，系统完成学员与教练一对一绑定 → 同步更新学员状态为【已分配教练，待学习】→ 学员端展示教练信息，教练端新增对应学员台账 → 支持管理员后续解绑、重新绑定操作，全程记录变更日志。
4.4 学习进度管理业务流程
学员与教练绑定成功 → 学员查看教练档期，自主预约练车时间，或取消未开始预约 → 教练根据预约安排教学，课后录入学员练车学时、练课记录 → 系统自动累计学时，实时更新学员学习阶段进度 → 学员依次完成科目一、科目二、科目三、科目四学习与学时达标 → 单个阶段达标后自动解锁下一阶段权限 → 全部学习阶段完成后，进入考试报名环节。
4.5 考试报名与成绩管理流程
学员对应科目学时、学习阶段达标 → 系统开放该科目考试报名权限 → 学员选择考试科目、考试时间、考场，提交报名申请 → 管理员审核报名资格，审核通过则报名生效 → 学员参加对应科目考试 → 考试结束后管理员录入考试成绩 → 成绩合格：自动解锁下一科目学习与报名权限；成绩不合格：系统生成补考记录，支持预约补考、登记补考信息 → 学员重复上述流程完成所有科目考试 → 全部科目合格后，系统触发证件发放提醒，管理员登记发证信息，完成学车全流程。
4.6 数据统计分析业务流程
系统实时采集报名、学习、考试、教练教学全量数据 → 按日/周/月/年自动汇总数据 → 分别统计报名人数、各科目考试通过率、教练工作量与带教数据 → 生成可视化图表与数据报表 → 管理员可自定义筛选时间、维度查询数据 → 支持报表导出、打印、存档，为驾校运营决策提供数据支撑。
